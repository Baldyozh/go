Программное средство предназначено для обработки, хранения, поиска и анализа логов интеграционных запросов. Система позволяет находить логи по идентификатору запроса, фильтровать события, просматривать детали лога, экспортировать данные и выполнять расшифровку чувствительных полей при наличии соответствующих прав.

6.1 Установка и настройка

Для установки программного средства необходимы:

Go версии 1.25.5 или выше;
Docker;
Docker Compose;
доступ к исходному коду проекта.
Порядок установки:

Перейти в каталог проекта:
Запустить инфраструктуру:
docker compose up -d
Проверить, что запущены сервисы:
Kafka;
Zookeeper;
Vault;
ClickHouse;
PostgreSQL;
pgAdmin.
При необходимости изменить настройки в файле:
config/config.yaml
Основные параметры:

адрес Kafka;
имя топика;
адрес Vault;
список полей для шифрования;
параметры подключения к PostgreSQL;
параметры подключения к ClickHouse;
порт API-сервера.
Запустить API-сервер:
go run cmd/api/main.go
Для запуска обработчика логов выполнить:
go run cmd/addlogs/main.go
6.2 Авторизация пользователя

Для входа в систему пользователь отправляет запрос:

POST /api/auth/login
Пример тела запроса:

{
  "username": "admin",
  "password": "admin123"
}
После успешной авторизации система возвращает JWT-токен. Его необходимо передавать в заголовке последующих запросов:

Authorization: Bearer <token>
6.3 Поиск логов по идентификатору запроса

Для поиска логов по конкретному запросу используется метод:

GET /api/logs/request/{request_id}
Порядок действий пользователя:

Получить JWT-токен.
Указать идентификатор запроса.
Отправить запрос к API.
Просмотреть список найденных логов.
При необходимости открыть детальную информацию по конкретному логу.
6.4 Фильтрация логов

Для фильтрации логов используется метод:

GET /api/logs
Доступные параметры:

request_id;
integration_id;
start_time;
end_time;
status_code;
http_method;
endpoint;
user_id;
success.
Пример запроса:

GET /api/logs?integration_id=partner1&status_code=500&success=false
Порядок действий пользователя:

Выбрать параметры фильтрации.
Отправить запрос.
Получить список логов.
Проанализировать найденные записи.
6.5 Просмотр детальной информации о логе

Для просмотра конкретного лога используется метод:

GET /api/logs/{id}
Пользователь получает:

идентификатор лога;
дату и время события;
идентификатор интеграции;
идентификатор запроса;
HTTP-метод;
endpoint;
тело запроса;
тело ответа;
код ответа;
длительность выполнения;
признак успешности;
сообщение об ошибке.
6.6 Расшифровка чувствительных данных

Если пользователю необходимо получить доступ к конфиденциальным данным, используется параметр decrypt=true:

GET /api/logs/{id}?decrypt=true&reason=Расследование инцидента
Расшифровка доступна только пользователям с правом logs:decrypt.

Порядок действий:

Открыть нужный лог.
Указать необходимость расшифровки.
Заполнить причину расшифровки.
Отправить запрос.
Получить расшифрованные данные при наличии прав.
Если у пользователя нет соответствующего права, система запрещает операцию.

6.7 Экспорт логов

Для экспорта логов используется метод:

GET /api/logs/export
Пользователь может передать те же фильтры, что и при обычном поиске. В результате система формирует CSV-файл.

6.8 Управление пользователями

Создание пользователей доступно только администратору.

Метод:

POST /api/admin/users
Пример тела запроса:

{
  "username": "newuser",
  "email": "newuser@example.com",
  "password": "securepassword",
  "role_ids": [1, 2]
}
Порядок действий администратора:

Авторизоваться под учетной записью администратора.
Указать имя пользователя, email, пароль и роли.
Отправить запрос.
Проверить, что пользователь создан.
6.9 Действия пользователя при сбое

Если API не отвечает:

Проверить, запущен ли API-сервер.
Проверить доступность контейнеров:
docker compose ps
Проверить логи контейнеров:
docker compose logs
Проверить настройки подключения в config/config.yaml.
Перезапустить инфраструктуру:
docker compose restart
Если пользователь не может авторизоваться:

Проверить правильность логина и пароля.
Убедиться, что пользователь существует в PostgreSQL.
Проверить, что миграции базы данных выполнены.
Обратиться к администратору системы.
Если не выполняется расшифровка:

Проверить наличие права logs:decrypt.
Убедиться, что указан параметр reason.
Проверить доступность Vault.
Обратиться к администратору безопасности.
